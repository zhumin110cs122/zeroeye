# Contributing to Tent of Trials

Thank you for your interest in contributing! This guide will help you set up your development environment and submit a quality pull request.

## Prerequisites

| Tool | Version | Purpose |
|------|---------|---------|
| Python | 3.10+ | Build orchestration (`build.py`) |
| Rust / Cargo | 1.70+ | Backend service |
| Node.js / npm | 18+ | Frontend |
| Go | 1.21+ | Market service |
| GCC / G++ | 12+ | Frailbox (C) and engine (C++) |
| CMake | 3.25+ | Engine build |
| Make | 4.3+ | Frailbox build |
| Java (JDK) | 17+ | Compliance auditor |
| Ruby | 3.1+ | V2 market stream |
| Lua | 5.4+ | NFC scanner, OpenAPI tools |
| GHC | 9.4+ | OpenAPI Haskell docs |

## Local Setup

```bash
git clone https://github.com/lobster-trap/zeroeye.git
cd zeroeye
python3 build.py
```

The build script compiles all modules and generates diagnostic artifacts in `diagnostic/`.

### Build specific modules

```bash
python3 build.py -m backend          # Rust only
python3 build.py -m frontend,market  # Multiple modules
python3 build.py --list              # List available modules
```

### Clean artifacts

```bash
python3 build.py --clean
```

## PR Workflow

1. **Fork** the repository on GitHub
2. **Branch** from `main`: `git checkout -b fix/your-feature`
3. **Commit** with clear messages
4. **Run** `python3 build.py` locally and verify all modules pass
5. **Push** to your fork: `git push origin fix/your-feature`
6. **Open a PR** against `main` using `.github/pull_request_template.md`

## Diagnostic Artifacts (Required)

Every PR **must** include the diagnostic artifacts generated by `build.py`:

- `diagnostic/build-XXX.logd` (encrypted build log)
- `diagnostic/build-XXX.json` (build metadata)

Include these files in your commit. The password for the `.logd` file is printed at the end of the build output and must be included in the PR description.

## Pre-Commit Hook

Install the pre-commit hook to automate diagnostic generation:

```bash
make install-hooks
```

See `docs/pre-commit-hook.md` for details.

## Code Style

This project uses `.editorconfig` for consistent formatting. Please ensure your editor respects these settings. Key conventions:

- **Python**: 4-space indent, type hints preferred
- **Rust**: `cargo fmt` default style (4-space indent)
- **TypeScript**: 2-space indent
- **C/C++**: 4-space indent, `-Wall -Werror -Wextra`
- **Go**: `gofmt` default (tabs)
- **Makefiles**: Tabs only
